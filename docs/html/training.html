Behavioural Training¶

Usage¶

Session objects represent a single training session and are first used to operate the hardware during a session. They are subsequently used to manipulate the data collected during that session for analysis.

Training data is stored in a JSON file; each mouse has a single JSON file containing all of its training data. At the top level, the JSON data is a list of dicts where each dict corresponds to the nth training day. This data is collated and manipulated together at the level of the Mouse, in the form of a list of Session objects.

A Mouse is instantiated from its corresponding training JSON by passing the mouse’s ID and the path to the folder containing the training JSONs:

mouse = Mouse.init_from_file(
    mouse_id='ExampleMouse1',
    json_path='~/training_data',
)

The Mouse.train method creates a new Session object and begins the training session:

mouse.train(
    config_file='~/reach_config.ini',
    data={'trainer': 'Matt', 'training_box': 1},
)

Here, we are passing the path to a config file which contains the training settings (more info below), and a dict containing any arbitrary data that we’d like to save alongside this session’s data.

Finally we save the training data back into the JSON file:

mouse.save_data_to_file('~/training_data')

There is an example script at scripts/run_session/py which uses argparse to take input from the user and run a training session.

Training settings¶

Configuration settings of a training session can be passed to Session.run(config) to change some of the task parameters. The Mouse.train() method can accept a str providing a path to a config file, which it will read settings from and use these to run a session:

mouse.train(
    config_file='~/reach_config.ini',
)

Default values will be used if it is not passed, or the file does not exist.

A config file with default values can be generated on demand:

reach.generate_config('~/reach_config.ini')
